# ClabGame 대시보드 요청사항 정리

## 1) 기본 방향
- 백엔드 없이 **프론트엔드만**으로 바로 실행 가능해야 함.
- 실제 게임 엔진 구현이 아니라, 오프라인/현실에서 진행된 결과를 기록하고 시각화하는 페이지.

## 2) 팀/초기 자금
- 팀 추가 기능 제공.
- 사용 팀 수는 보통 2~3개 (최대 3개).
- 팀 생성 시 입력값
  - 팀 이름
  - 팀 한마디
- 초기 Seed는 팀당 고정 **1억**.

## 3) 이벤트 입력/회차
- 팀 선택 단일 입력이 아니라, **각 팀마다 독립 입력 카드** 제공.
- 팀 카드별 입력 요소
  - 이벤트 유형 선택
  - 금액 입력
  - 메모 입력
  - 팀별 기록 버튼
- 회차는 수동 선택이 아닌, **팀별로 입력할 때마다 자동 증가**.
- 이전 기록은 실수 수정이 가능해야 함 (수정/삭제).

## 4) 이벤트 유형
- 일반 유형 목록에서 선택.
- 추가 유형 포함
  - 빅딜
  - 사내 단합 게임

### 빅딜 규칙
- 팀이 빅딜을 선택하면 상대팀/승패를 지정.
- 승리 시: 두 팀의 현재 자금을 서로 교환.
- 패배 시: 변화 없음.
- 상대팀 영향도 기록에 반영.

## 5) 금액 단위 정책
- 직접 입력하는 금액: **만원 단위**.
- 이벤트 증감(기록 테이블의 변동 금액): **만원 단위**.
- 누적 자금/현재 자금/그래프: **억 단위** 표기.

## 6) 화면 구성
- 현재 자금 현황
- 이벤트 이력 테이블 (회차, 팀, 유형, 금액, 누적, 메모, 수정/삭제)
- 회차별 자금 그래프
- 현재 자금 기준 리더보드
  - 금액 표시
  - 팀 한마디 표시
  - 리더보드에서 팀 한마디 수정 가능

## 7) 저장/초기화
- 새로고침해도 데이터 유지 (예: localStorage).
- 전체 리셋 버튼으로 데이터 초기화.

## 8) 디자인
- 전반적으로 파스텔 톤 중심의 보기 좋은 UI.
